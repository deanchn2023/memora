# 忆境 Memora：用 ADP 打造你的专属 AI 个人助理

> 从剪贴板监听到知识图谱，从任务管理到投标助手——一个普通人如何用腾讯云 ADP 智能体平台，构建出越用越懂你的 AI 伙伴。

---

## 一、痛点：为什么我需要一个"第二大脑"？

作为一个同时负责产品、技术、客户的"多线程打工人"，我每天面对的信息量是惊人的：

- **微信群消息**：客户需求、内部讨论、项目更新——刷过就忘
- **会议纪要**：开完会一堆待办，转头只记得大概
- **文档资料**：技术文档、投标模板、产品白皮书——找的时候翻半天
- **碎片想法**：灵感总在最不方便记录的时候冒出来

更痛苦的是，这些信息散落在各个工具里——微信、飞书、邮件、浏览器书签——没有一个地方能把它们串起来。我开始思考：

> **如果有一个工具，能自动记住我看过什么、聊过什么、做过什么，还能在我需要的时候主动提醒和推荐，那该多好？**

---

## 二、初识 ADP：原来做一个 AI 应用这么简单

腾讯云 ADP（智能体开发平台）让我眼前一亮——不需要从零训练模型，不需要搭建复杂的 AI 基础设施，**只需要创建一个 Claw 应用，配置好知识库和 Prompt，就能拥有一个懂业务、能对话、会干活的 AI 助手**。

在 ADP 上创建应用的流程非常简单：

1. **登录 ADP 控制台** → 点击"创建应用"
2. **选择应用类型** → 配置名称、描述、头像
3. **上传知识库** → 把产品文档、技术规范、FAQ 等资料喂给它
4. **编写系统 Prompt** → 告诉 AI 它的角色、能力边界和回复风格
5. **获取 AppKey** → 一行代码就能在客户端调用

[截图：ADP 创建应用流程]

最关键的是，ADP 提供了 **SSE 流式输出**——对话内容实时推送，用户不需要干等，体验跟 ChatGPT 一样流畅。而且 ADP 支持多种模型（混元、DeepSeek、Kimi 等），可以根据场景选择最合适的模型。

## 三、Memora 的诞生：让 AI 真正融入日常工作

有了 ADP 提供的 AI 能力底座，我开始思考：怎么让 AI 不只是一个"对话框"，而是真正融入日常工作流的"伙伴"？

答案是——**让 AI 主动工作，而不是被动回答**。

### 3.1 智能剪贴板：你复制什么，它就懂什么

Memora 最核心的创新是**剪贴板智能感知**。当你在工作中复制一段文字——可能是微信里的客户需求、邮件里的项目安排、网页上的技术文档——Memora 会自动分析内容：

- **识别为待办事项** → 弹窗建议创建任务，自动提取标题、优先级、截止日期
- **识别为知识信息** → 自动提取为结构化记忆，按人物/项目/技术分类
- **识别为学习资料** → 保存到记事本，支持后续整理

[截图：剪贴板弹窗]

这意味着——你只需要正常工作，Memora 就在默默帮你记录和整理。不需要刻意打开 App、不需要手动输入，信息自然沉淀。

### 3.2 三层记忆系统：像人脑一样存储和遗忘

不是所有信息都一样重要。Memora 采用了类似人脑的三层记忆架构：

| 记忆层级 | 保留时间 | 典型内容 | 示例 |
|---------|---------|---------|------|
| **瞬时记忆** | 5分钟~1小时 | 当前工作上下文 | "正在看 ADP V2 接口文档" |
| **短期记忆** | 1天~7天 | 近期关注、项目、人物 | "康院是北汽福田的客户，在做 FMEA 项目" |
| **长期记忆** | 数月~永久 | 长期目标、核心关系 | "强总是我的直属上级，关注 ADP 产品方向" |

AI 会自动判断信息的时效性，让重要的记忆长期保留，临时信息自然消退。而且这些记忆不是孤立的——Memora 会构建**实体关联图**，把人物、项目、公司串联起来，形成你的个人知识网络。

> 比如当 AI 再次看到"康院"这个名字，它自动知道：北汽福田客户、FMEA 项目负责人、之前讨论过智能体方案。这就是"越用越懂你"的秘密。

[截图：记忆系统界面]

### 3.3 智能记事本 + 番茄钟 + 日历：工具链闭环

Memora 不只是记录，还帮你**行动**：

- **记事本**：AI 自动分类笔记，一键转为待办或提炼记忆
- **番茄钟**：25 分钟专注工作，关联具体任务追踪完成度
- **日历视图**：日/周/月三维视图，任务自动同步到 macOS 系统日历

## 四、ADP 赋能：从"助手"到"专家"

如果说 Memora 的本地能力是"记忆和执行"，那 ADP 提供的就是"思考和专业知识"。两者的结合，让 Memora 从一个个人效率工具，升级为真正的**工作伙伴**。

### 4.1 知识跟随：剪贴板感知 → 主动知识推荐

这是 Memora 和 ADP 结合最精彩的部分——**知识跟随**。

想象这个场景：你在微信群里看到一个同事分享了 ADP V2 接口调用的经验，你复制了那段文字。Memora 不只是帮你保存笔记，它还会：

1. **识别意图**：检测到内容涉及"API 接口""调用规范"→ 判定为"搜索知识"意图
2. **调用 ADP**：自动把相关关键词发给 ADP 智能体
3. **推送推荐**：ADP 基于知识库返回相关文档、技术规范、最佳实践
4. **保存沉淀**：推荐的知识一键保存到本地知识库

```
剪贴板内容 → 意图识别 → ADP 智能搜索 → 知识推荐 → 保存到本地
```

这意味着——**你不用主动搜索，知识会主动找到你**。你在看什么、关注什么，ADP 就推荐什么。这才是"知识跟随"的真正含义。

[截图：知识跟随推荐界面]

### 4.2 智能任务分析：结合本地上下文的 AI 排程

Memora 内置了一个 **Local Agent**，它能读取你的所有任务、记事本、记忆数据，结合 ADP 的推理能力，做到真正的"智能排程"：

- **今日排程**：分析所有待办的重要程度、截止日期，推荐最优工作顺序
- **生成日报**：自动汇总今天的任务完成情况，生成 Markdown 格式日报
- **整理笔记**：把碎片化的笔记按主题归档，发现重复内容
- **紧急事项提醒**：识别被遗漏的高优先级任务，主动提醒

和其他 AI 助手不同的是，Memora 的分析不是凭空猜测——它**真的知道你今天做了什么、还有哪些没做**，因为所有数据就在本地。

[截图：AI 助手对话 - 今日排程]

### 4.3 投标助手：ADP 知识库的杀手级应用

作为 ADP 产品的负责人，投标是我工作的重要部分。传统投标流程中，技术偏离表、SOW、验收标准……这些文档写起来耗时耗力，而且每次都要翻以前的模板。

现在，通过 ADP 上传产品文档、投标模板、历史标书作为知识库，Memora 成了我的**专属投标助手**：

| 快捷功能 | ADP 知识库来源 | 输出 |
|---------|-------------|------|
| 生成技术偏离表 | 产品功能清单 + 招标要求 | 标准化偏离对比表 |
| 生成技术标书 | 产品白皮书 + 架构文档 | 结构化技术方案 |
| 生成投标方案 PPT | 产品介绍 + 客户案例 | 大纲和关键页内容 |
| 生成点对点应答 | 招标文件 + 产品能力 | 逐条响应文档 |
| 生成 SOW | 项目模板 + 实施规范 | 工作说明书 |
| 生成验收标准 | 产品文档 + 行业标准 | 验收条款清单 |
| 生成私有化部署方案 | 部署手册 + 架构图 | 部署架构和步骤 |

这些功能背后，ADP 的知识库理解能力是关键——它不是通用 AI 在瞎编，而是**基于你上传的真实产品文档来回答**，准确性和专业性有保障。

### 4.4 知识助手：产品知识随时查

日常工作中，客户和同事随时可能问到产品细节：

- "ADP 支持哪些模型？"
- "最新版本有什么新功能？"
- "私有化部署有什么要求？"

通过 ADP 知识库，这些问题的答案触手可及——在 Memora 里输入问题，ADP 基于知识库秒级返回准确答案，不再需要翻文档或问同事。

## 五、技术架构：简单但不简陋

对于技术同学，简单说一下 Memora 的架构选择：

| 层级 | 选型 | 理由 |
|------|------|------|
| 桌面框架 | Electron | 跨平台 + 访问剪贴板/文件系统等系统 API |
| 前端 | 原生 HTML/CSS/JS | 轻量、无需构建、启动快 |
| 本地 AI | DeepSeek API | 兼容 OpenAI SDK、性价比高 |
| 智能体 | 腾讯云 ADP | 知识库管理 + 多模型 + SSE 流式输出 |
| 数据存储 | lowdb + JSON 文件 | 零配置、数据本地可控 |

核心设计原则：**数据在本地，智能在云端**。

- 敏感数据（任务、记忆、笔记）全部存储在本地，不上传任何服务器
- AI 能力按需调用——剪贴板分析、知识搜索、投标助手等走 ADP
- 配置管理支持组织级别——管理员统一下发 API Key、ADP AppKey、Prompt 模板

## 六、从 0 到 1：一个普通人的 AI 应用构建之路

回顾整个构建过程，我想分享几个关键决策：

### 6.1 先解决自己的问题，再想通用化

Memora 的每一个功能都是我自己在用的。剪贴板监听是因为我总忘事，知识跟随是因为我找不到文档，投标助手是因为我厌倦了重复劳动。**做自己的第一个用户**，让产品方向始终清晰。

### 6.2 ADP 大幅降低了 AI 应用门槛

如果没有 ADP，我需要自己搭建向量数据库、管理知识库索引、对接多种 AI 模型、实现流式输出……光是 AI 基础设施就要花几周。而 ADP 让我把精力集中在**产品体验和业务逻辑**上，AI 能力一行 AppKey 就搞定。

### 6.3 本地 + 云端的混合架构

完全本地太受限（没有 AI 能力），完全云端不安全（数据上传）。Memora 的混合架构让两者各司其职：本地负责数据管理和隐私保护，云端（ADP）负责智能推理和知识检索。

### 6.4 让 AI 学会自我进化

Memora 记录了用户的每一次反馈——接受、拒绝、修改——这些数据用于动态优化 Prompt 和 Few-shot 示例。用的越久，AI 对你的理解越精准，这比任何通用 AI 助手都更贴合个人需求。
