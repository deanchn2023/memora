## Bug 修复：富文本编辑器链接按钮点击无响应
- **根因**：`richEditor.js` 中 `prompt()` 在 Electron 渲染进程默认返回空字符串/被阻止，导致链接永远无法创建
- **修复**：替换 `prompt()` 为自定义 Apple Design 模态弹窗 `_showLinkDialog()`，包含 URL 输入框和链接文字输入框
- **关键**：弹窗前保存选区 `Range.cloneRange()`，插入链接前恢复选区，否则 `createLink` 作用在错误位置
- **支持**：Enter 确认、Escape 取消、点击遮罩关闭

## Bug 修复：AI 智能分析富文本保存到记事本丢失图片和格式
- **根因**：`src/scripts/notebook.js` 的 `addNote()` 方法创建 newNote 对象时没有包含 `htmlContent` 字段，导致前端传入的富文本 HTML（含 base64 图片）被丢弃，只保存了纯文本 `content`
- **修复**：在 `addNote()` 的 newNote 对象中添加 `htmlContent: note.htmlContent || null`
- **影响范围**：仅 `notebook.js` 一行改动，前端代码已正确传递 `htmlContent`，显示端 `loadNotes()` 也已优先使用 `htmlContent` 渲染

## 评估：Qwen2.5:0.5b 本地模型嵌入 → 放弃
- 0.5b 模型做完整结构化 JSON 分类（14字段）输出质量差（is_task判错、时间解析乱编、priority双值）
- 简化为二元判断（is_task + is_valid_info）虽可行，但投入产出比低：400MB模型 + node-llama-cpp原生编译 + 硬件兼容，只为省80%分类Token不值得
- 结论：放弃本地模型方案，等更强模型（7b量化/MLX框架）成熟后再考虑

## 功能完善：数据同步支持 htmlContent（富文本）
修改了 3 个文件的 6 个位置：

**服务端 `config-server/sync-routes.js`**：
1. `user_notes` 表加 `html_content TEXT DEFAULT ''` 字段
2. `DATA_TYPE_FIELDS.notes` 白名单加 `html_content`

**客户端 `main.js` 推送**：
1. `fullSync` 笔记映射加 `html_content`（含 base64 剥离）

1. `addNote` 加 `htmlContent: note.html_content`

**关键设计**：base64 内嵌图片在同步时上传到服务器并替换为服务端 URL（而非之前简单剥离为占位符），跨设备可访问。拉取时本地有 base64 版本则优先保留（file:// 页面加载 http:// 图片可能有问题），但同步推送后也会将 URL 版本写回本地笔记。

## Bug 修复：async map 中使用 await 导致 SyntaxError
- **错误**：`main.js:4450` 在 `.map(n => {...})` 回调中使用 `await processHtmlContentForSync()`，但回调不是 async 函数
- **修复**：`.map(n => {` → `.map(async n => {`，同时 `data.changes.notes = changedNotes.map(...)` → `data.changes.notes = await Promise.all(changedNotes.map(...))`
- **关键**：async map 返回 Promise 数组，必须用 `await Promise.all()` 等待所有异步映射完成，否则赋值的是 Promise 数组而非结果数组

## Bug 修复：富文本图片同步后不显示
- **根因**：同步推送时 `processHtmlContentForSync` 将 base64 上传到服务器并替换为 URL，但结果仅用于发送不写回本地。同时拉取时服务端的 URL 版本可能覆盖本地 base64 版本，而 `file://` 页面加载 `http://` 图片可能失败
- **修复1**：推送上传成功后，将 URL 版本写回本地笔记 `notebook.updateNote(n.id, { htmlContent: htmlContentForSync })`
- **修复2**：拉取时 `resolvedHtmlContent` 逻辑改为本地有 htmlContent 则优先保留本地版本
- **验证**：服务器静态文件路由 `/memora/uploads/note-images/` 已配置（app.js:69），curl 本地和外部均可 200 OK 访问图片
