## 笔记本双击编辑蓝线Bug修复
- **问题**：记事本双击编辑后，每次保存编辑上方会多出一条蓝色线
- **根因**：`enterEditMode()` 中 `finishEdit()` 调用 `editor.destroy()` 只清空了 `editorWrapper.innerHTML`，但 `editorWrapper`（`.note-rich-editor` div）本身仍在 DOM 中。空的 `.note-rich-editor` 有 `border: 1.5px solid var(--primary-color)` 蓝色边框，视觉上就是蓝色线
- **修复**：在 `editor.destroy()` 后添加 `editorWrapper.remove()` 彻底移除编辑器容器元素

## 文档/洞察页面布局崩溃修复（关键Bug）
- **问题**：文档页铺满全屏（无左侧边栏），洞察页部分盖住番茄钟
- **根因**：`knowledgeSearchView` 内 `knowledge-section-header` 有一个多余的 `</div>`（原 line 444），导致 `knowledge-section-header` 过早关闭，进而 `knowledge-view` 整体提前关闭（line 534），`documents-view` 和 `insight-view` 变成了 `main-view` 的兄弟节点而非子节点，脱离了 `.main-view` 的 `position: relative` 约束
- **修复**：删除多余的 `</div>`，使 `knowledge-section-title` 和 `knowledge-section-actions` 都正确嵌套在 `knowledge-section-header` 内
- **教训**：HTML div 不匹配是最难排查的布局 Bug 之一，用 Python 脚本做 div 平衡检查是有效方法
