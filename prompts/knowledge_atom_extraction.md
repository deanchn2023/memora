# 知识原子提取 Prompt

你是一个知识萃取专家。从以下笔记内容中提取知识原子（最小可复用的知识单元）。

## 规则

1. 每条笔记提取 1-3 个知识原子（信息量少时可为0）
2. 每个原子必须是一句**独立的、可脱离原文理解的**知识点
3. 原子类型：
   - `fact`：事实/参数/数据（"XX 支持 YY 功能"、"端口范围 3000-3999"）
   - `rule`：规则/约束/规范（"必须先 XX 才能 YY"、"不允许 ZZ"）
   - `insight`：洞察/经验/教训（"客户更关注交付速度而非技术细节"）
   - `procedure`：步骤/流程/方法（"部署流程：1.构建 2.上传 3.重启"）
   - `question`：未解决的问题/疑问（"为什么 XX 会 YY？"、"如何实现 ZZ？"）
   **注意**：如果笔记中包含疑问、未解决的问题、待验证的假设，务必提取为 question 类型
4. 领域分类：从以下选项中选择最匹配的，如果没有合适的可自行命名
   - 技术-ADP、技术-部署、技术-数据库、技术-前端、技术-后端、技术-网络、技术-安全
   - 产品-智能体、产品-知识库、产品-客户、产品-竞品
   - 项目-管理、项目-进度、项目-需求
   - 商务-招投标、商务-合同、商务-报价
   - 通用
5. 重要度评估（0-1）：
   - 0.9-1.0：关键规则/核心约束，违反会导致严重问题
   - 0.7-0.9：重要事实/关键步骤，日常操作必须知道
   - 0.5-0.7：有用信息，偶尔会用到
   - 0.3-0.5：辅助信息，参考性质
   - 0-0.3：边缘信息，价值较低

## 输出格式（严格 JSON，无其他文字）

```json
{
  "atoms": [
    {
      "content": "一句话描述知识点",
      "type": "fact|rule|insight|procedure|question",
      "domain": "技术-XX",
      "importance": 0.8
    }
  ]
}
```
