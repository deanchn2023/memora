# 忆境 Memora v2.0 — AI 能力说明

> **你的个人 AI 记忆助理，越用越懂你**

Memora v2.0 是一个深度集成 AI 的桌面应用，AI 不是附加功能，而是系统运转的核心引擎。从你复制的一段文字开始，到知识的沉淀与唤醒，AI 全程参与、持续学习、自我优化。

---

## 一、AI 能力全景

Memora 的 AI 能力覆盖 **感知 → 理解 → 搜索 → 沉淀 → 决策** 五大环节，形成完整的智能闭环：

```
┌─────────────────────────────────────────────────────────────┐
│                    Memora AI 能力矩阵                        │
├──────────┬──────────────────────────────────────────────────┤
│  感知层   │ 剪贴板实时监听 · 意图识别 · 待办提取 · 信息过滤   │
│  理解层   │ 语义记忆提取 · 实体识别 · 知识原子化 · 自动聚类    │
│  搜索层   │ 关键词提炼 · ADP语义搜索 · 知识推荐 · 本地检索     │
│  沉淀层   │ 知识簇合成文章 · 记忆分层管理 · 番茄钟专注跟踪     │
│  决策层   │ 优先级排程 · 日报周报 · 多意图Agent · 用户画像     │
└──────────┴──────────────────────────────────────────────────┘
```

## 二、核心 AI 功能详解

### 2.1 剪贴板智能感知 — 无感录入，精准识别

**AI 在这里做什么**：当你复制任何文字，AI 在 3 秒内完成三重判断：

| 判断维度 | AI 输出 | 举例 |
|---------|--------|------|
| **待办识别** | `is_task: true/false` + 优先级 + 截止时间 | "@Dean 你整理一下方案" → 待办，高优先级 |
| **信息有效性** | `is_valid_info: true/false` + 分类 + 摘要 | 产品规格参数 → 有效信息，保存 |
| **知识推荐** | `needs_recommendation: true/false` + 意图 | "这个方案怎么接入？" → 需要推荐，触发搜索 |

**数据流程**：
```
剪贴板变化 → 文本去重 → AI 意图识别（DeepSeek）
    ├→ 待办任务 → 任务列表（自动设优先级、关联人物/项目）
    ├→ 有效信息 → 记事本（自动分类、提取摘要）
    ├→ 知识推荐 → ADP 知识搜索（语义检索相关知识）
    └→ 无效内容 → 静默丢弃（闲聊、确认语、情绪表达）
```

**为什么准确**：AI 不仅分析文本本身，还参考你的用户画像（高频人物、活跃项目、优先级偏好），让判断更贴合你的工作场景。

---

### 2.2 语义记忆提取 — 从文字到结构化记忆

**AI 在这里做什么**：将碎片文本转化为结构化记忆，自动完成：

- **记忆分层**：瞬时（instant）→ 短期（short）→ 长期（long），AI 根据内容语义判定
- **内容分类**：task / interest / person / project / goal / knowledge / action
- **实体提取**：识别人物、公司、产品、技术、项目等实体，自动关联已知实体库
- **摘要生成**：50 字以内精准摘要
- **关联发现**：自动匹配高频人物和活跃项目

**数据流程**：
```
原始文本 → 记忆提取AI（DeepSeek + 用户画像 + 已知实体库）
    ├→ memory_type: instant/short/long（基于语义判定）
    ├→ category: 自动分类
    ├→ entities: 实体提取（复用已知ID，避免重复）
    ├→ linked_known_persons: 关联高频人物
    ├→ linked_known_projects: 关联活跃项目
    └→ ttl_hint: 建议过期/晋升时间
```

---

### 2.3 知识萃取引擎 — 从碎片到体系

这是 Memora 最核心的 AI 能力，将你的碎片化笔记自动提炼为结构化知识体系：

#### 三阶段提炼流程

```
阶段一：知识原子提取
笔记内容 → AI 提取最小知识单元（fact/rule/insight/procedure/question）
         → 每个原子独立可检索、可复用

阶段二：AI 智能聚类
未归簇原子 + 已有簇 → AI 按主题智能分组
                    → 优先归入已有簇，必要时新建
                    → 自动合并相似簇、清理空簇
                    → 簇成熟度判定（growing → mature）

阶段三：知识文章合成
成熟簇（≥5个原子）→ AI 合成结构化 Markdown 文章
                   → 每段标注来源原子，可溯源
                   → 自动更新簇状态为 distilled
```

**一键萃取**：串联执行 合并相似 → 清理空簇 → AI 聚类 → 生成文章

---

### 2.4 知识搜索与推荐 — 语义检索 + 多源融合

**AI 在这里做什么**：

1. **关键词提炼**：将自然语言查询提炼为核心关键词（≤7字），适配各搜索源
2. **ADP 语义搜索**：调用腾讯 ADP 智能体，理解查询意图，返回精准知识
3. **本地记忆检索**：在记忆库中关键词匹配
4. **公开资源搜索**：在知识图谱、文章库中检索

**数据流程**：
```
用户搜索/剪贴板疑问
    → AI 提炼核心关键词（DeepSeek）
    → 并行搜索三路：
    ├→ ADP 语义搜索（knowledge_app_key → 知识推荐智能体）
    ├→ ADP 搜索问答（search_app_key → 搜索问答智能体）
    ├→ 本地记忆搜索（关键词匹配）
    └→ 公开资源搜索（图谱 + 文章）
    → 结果聚合展示
```

---

### 2.5 多意图 Agent 系统 — 一个入口，五个专家

**AI 在这里做什么**：用户对话时，AI 自动分类意图，调用对应专家 Agent：

| Agent | 意图 | 能力 |
|-------|------|------|
| **Priority Agent** | priority | 综合截止时间、优先级偏好、工作时段，为今日排程 Top 5 |
| **Knowledge Agent** | knowledge | 碎片笔记聚类、合并建议、洞察提取 |
| **Memory Agent** | memory | 记忆晋升/降级/淘汰/合并建议 |
| **Report Agent** | report | 自动生成日报/周报，含任务完成情况、洞察、明日建议 |
| **Chat Agent** | chat | 上下文感知通用对话（注入记忆、任务、笔记） |

**RAG 检索增强**：每个 Agent 调用前自动检索相关上下文（记忆、笔记、任务、实体），让 AI 回答更精准。

---

### 2.6 ADP 智能体集成 — 企业级知识库对话

集成腾讯 ADP V2 智能体平台，支持：
- **流式 SSE 对话**：实时渲染 AI 回复
- **三密钥体系**：通用对话 / 知识推荐 / 搜索问答 使用不同智能体
- **结果自动保存**：AI 回复自动归入知识推荐列表
- **会话历史管理**：多轮对话上下文保持

## 三、AI 自学习与自我优化机制

> **Memora 不只是用 AI，更让 AI 越用越准**

### 3.1 反馈闭环系统（Feedback Loop）

这是 Memora AI 持续优化的核心机制：

```
AI 输出 → 用户操作（接受/删除/修改）
    → 反馈记录（trace_id + 输入 + AI输出 + 用户最终结果 + 正/负标签）
    → 注入后续 Prompt 的正/负样本
    → AI 下次遇到相似输入时参考历史判断
```

**具体实现**：
- 每次AI调用生成唯一 `trace_id`，记录完整调用链
- 用户删除AI生成的记忆 → 记为**负样本**（含删除原因）
- 用户保留/修改AI生成的记忆 → 记为**正样本**（含用户最终版本）
- 后续 Prompt 自动注入最近 N 条正/负样本作为 few-shot 示例

**效果**：AI 会记住"你删过什么"和"你改过什么"，逐步学会你的偏好。

### 3.2 用户画像自适应

系统持续学习用户行为，构建动态画像：

| 维度 | 数据来源 | AI 影响 |
|------|---------|---------|
| 高频人物 | 剪贴板/笔记中出现的人名频率 | 影响记忆分层（命中高频人物 → 升级为长期记忆） |
| 活跃项目 | 任务/笔记中关联的项目 | 影响待办识别（涉及活跃项目 → 提高优先级） |
| 优先级偏好 | 用户的优先级调整历史 | 影响新任务的优先级判定 |
| 工作时段 | 任务完成时间分布 | 影响排程建议（在高效时段安排重要任务） |
| 行业/角色 | 用户设定 + 内容语义 | 影响分类和摘要的专业度 |

### 3.3 记忆分层自动演进

```
瞬时记忆（instant）     ← 当前工作上下文，5分钟~1小时
    ↓ 多次访问 / 关联高频人物
短期记忆（short）       ← 近期项目动态，1天~7天
    ↓ 反复引用 / 包含决策目标
长期记忆（long）        ← 核心知识，数月
    ↓ 3个月未访问 / 内容过时
短期记忆（降级）        ← 自动降级或淘汰
```

AI 会定期检查记忆，建议晋升/降级/淘汰/合并，让记忆库始终保持有效和精简。

### 3.4 Prompt 持续迭代

- 所有 AI Prompt 均为可编辑的 Markdown 模板（`prompts/` 目录）
- 支持在线编辑 Prompt，实时生效
- 支持远程配置下发统一 Prompt，确保组织内一致
- 模板变量自动填充用户画像、上下文、正负样本

## 五、AI 调用模型与接口

| 场景 | 模型/接口 | 说明 |
|------|----------|------|
| 意图识别、记忆提取、关键词提炼、知识萃取 | **DeepSeek** (`deepseek-v4-flash`) | 低延迟、高性价比，适合结构化提取任务 |
| 语义搜索、知识推荐 | **ADP V2** (knowledge_app_key) | 企业知识库问答，SSE 流式返回 |
| 搜索问答 | **ADP V2** (search_app_key) | 搜索增强问答，更精准的检索结果 |
| 通用对话 | **ADP V2** (app_key) | 通用 AI 助手对话 |
| 多意图 Agent | **DeepSeek** + RAG 上下文注入 | 意图分类 + 专家 Prompt + 结构化输出 |

所有 AI 调用均支持远程配置切换模型和密钥，组织管理员可通过配置服务统一管理。

## 六、隐私与安全

- **数据本地优先**：所有记忆、笔记、知识原子、文章均存储在本地（SQLite + JSON），不上传云端
- **AI 调用最小化**：只在需要智能分析时调用 AI，基础操作（增删改查）无需 AI
- **反馈数据本地存储**：正/负样本日志存储在本地 `feedback/` 目录，不外传
- **Token 安全**：API Key 通过环境变量或远程配置注入，不硬编码
- **组织配置隔离**：不同组织的 ADP 密钥、Prompt 模板完全隔离

## 七、技术架构

```
Electron (渲染进程 + 主进程)
  ├── 前端：HTML/CSS/JS（Apple Design Language）
  ├── 主进程：IPC 通信 + AI 调用 + 数据管理
  ├── AI 引擎：DeepSeek API + ADP V2 SSE
  ├── 数据层：SQLite (better-sqlite3) + JSON 文件
  ├── Prompt 引擎：Handlebars 模板 + 动态变量注入
  ├── 反馈系统：trace 日志 + 正/负样本自动注入
  └── 配置服务：远程配置 + 本地配置双通道
```
